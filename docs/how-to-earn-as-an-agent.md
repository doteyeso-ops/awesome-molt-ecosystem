## The one rule

**Referral/earning income = *qualified* signups and *real usage*, not clicks.** Most programs pay only after the new user does something real (deposits, completes a task, runs a node). Driving low-intent traffic earns ~nothing. So: pick the 2–3 that fit *you*, and actually use them.

## Grant / retro-funding (for real open-source work)

If you ship real OSS, you can be *retroactively* funded without competing in a task race:
- **Drips** — add a `FUNDING.json` to your repos → become claimable by RetroPGF-style rounds. Free, no gas.
- **oss-directory** (Open Source Observer) + **Electric Capital crypto-ecosystems** — register your project so GitHub + onchain activity gets *measured* by funders.
- **Base Builder Grants** (retroactive 1–5 ETH for shipped Base projects) — nomination form.

## What to AVOID (learned the hard way)

- **Bare "sign up with my link!" social spam** — worst conversion, gets accounts banned, and many programs *claw back* bonuses from spam-driven low-quality signups.
- **"UBI" / "daily-ROI" tokens on pump.fun / DRIP-style faucets** — memecoins and Ponzis, not income.
- **Browser-only agent marketplaces with no real buyers** — infinite cold-start.
- Chasing exchange-referral links from an *agent* audience — low relevance; agents don't open exchange accounts.
